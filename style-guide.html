Preface (permalink) (go to top)

This page is/was used to check how elements and styling behave and look. My muses are innumerable, but some of the more important ones that deserve a citation are:

  • XXIIVV

    Some things that I really liked about this design is its simplicity and its use of images to make a barebones site look animated. I decided to go with their style of navigation, with vertical links of two columns. I think it is fairly obvious how much my site shares with this site.

  • Gwern

    Although I disagree with this site’s liberal use of Javascript to supposedly improve the reader experienceAnd make the site much slower as a result. Sometimes I wish that there was a Javascript-free version of Gwern so I do not have to go through the painful process of waiting for things to load. This is also a test for sidenotes, to see if they are displayed accurately. I’m using CSS pilfered from Github: Tufte CSS, since I think that it is effective and fast for my purposes., I still enjoy their prioritization of the content, making sure that the process of content to other content is as smooth as possible. I was not sure whether I should make body text justified or not, since I heard that the hyphenation algorithm for web browsers is not on-par with the TeX hyphenation algorithm, but after seeing their body text, I was sold on justification, even if it meant awkward hyphenation and unusual word spacing.

  • Brutalist Web Design

    I think there is beauty in efficiency, and this website convinced me to put content first, before anything else (among other nudges from various sites, most of which are on this list). I have tried to follow these guidelines, and if I have transgressed upon them, it is probably for a good reason.

  • Just Fucking Use HTML and Motherfucking Website

    These two websites helped me understand the uses for different tags, among their satirical jabbing at Javascript frameworks.Which I fully agree with. See Days Since Last JS Framework, which is arguably not long enough. How many frameworks do you really need? Plain HTML (without CSS) can be useful and beautiful in its own right, and by jumping straight to the heavy-hitters, we miss out on the strength of HTML, which may be better suited for a project.

  • The Garden of Maggie Appleton

    I think the idea of a digital garden is interesting, and this was one of the digital gardens which popped out for me the most. I don’t know how I’ll accidentally mimic this website, but I probably will.

  • Github: Tufte CSS

    I enjoy how this style looks so much that it is a given that my website will bear a heavy resemblance towards it. In particular, I am using its sidenote CSS, which is not as complex as I thought it would have been, a week ago when I didn’t know any HTML and CSS. It is really just the use of the float property.

  • This Page is Designed to Last: A Manifesto for Preserving Content on the Web

    Although occupying the same niche as Brutalist Web Design, it goes further in its spartan requirements, which is justified due to its mission: slow down internet rot.

  • Karthinks

    I especially enjoy how its table of contents is fixed to the right, allowing for easy access. That same feature is also in my website. Its distribution of information is also extremely streamlined, and I have enjoyed many good articles on specific Emacs functionalities from this site.

Notes about the sidenote/marginnote features (permalink) (go to top)

One feature that I especially like in other websites and in my own is sidenotes (and their close cousin/doppleganger, marginnotesIs it marginnote with no spaces or margin note with one space? Or is it margin-note with a hyphen separating these two words?), because it allows the writer to include notes that run in all sorts of directions away from the general purpose of the content in a way that is not tiring for the reader. Because I am using the Tufte CSS method for sidenotes, they are simple but easily broken. If they are placed too closeThe pushing problem in question. I don’t know if this is the case for all browsers though. to each other, they don’t overlap, but the lower sidenote is pushed down, separating it from its marker. I also follow the Tufte CSS advice in including figures only in marginnotes, as I have discovered that the multimedia in question will conflict with the number marker.

Marginnotes are the same as sidenotes, except they have no marker, which makes them more general than sidenotes. For a style guide on usage, I would imagine that I would use sidenotes to make opaque some witty pun that I thought of or to explain something in more depth that is ancillary to the purpose of the main content. For marginnotesYou can see how it is now alternating between sides, which allows for more compact notes. The image quality isn’t too good, but it is probably fine enough for a casual examination of the layout., I would imagine that I would use them to show work (for math and its parallel subjects) that the normal reader might find tedious to walk through but may be helpful for a newcomer (if I ever become skilled to the point where I’m writing math notes).

When printing out a page, I have made it so that every other sidenote/marginnote is placed on the left margin instead, which alleviates this pushing problem. However, this cannot be done on the website, since the extra space needed is occupied by the fixed table of contents, which I simply think is too useful to be replaced by left-margin notes.

Markup in text (permalink) (go to top)

There are many markup options available in-text, like bolding a word or two to emphasize it (one must make sure to never bold too many words however) or italicizing a word or two (not recommended because it is too light to be seen easily, unlike bolding). To indicate a series of keypresses, we have the <kbd> tag, so you can write M-x tetris. Links are designated by an underline. I wasn’t sure whether or not to also add color as a link quality, but decided against it, because I like how the website looks with monochromaticity.
